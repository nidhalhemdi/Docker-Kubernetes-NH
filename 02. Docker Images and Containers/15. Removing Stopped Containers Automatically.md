# **Removing Stopped Containers Automatically**

### **Key Idea**

Docker containers that stop remain on your system unless you manually remove them. Over time, this creates a long list of stopped containers.
To avoid this, Docker provides the `--rm` flag when running containers.

### **`--rm`**

Automatically removes the container as soon as it *exits/stops*.

You add it to `docker run`:

```bash

```

This means:

* You **do not** need to manually run `docker rm` later.

Use this flag when:

* You rarely need to restart a container.

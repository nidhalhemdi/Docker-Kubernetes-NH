# **Inspecting Images**

### **What an Image Actually Contains**

* A Docker image includes:

  * Your **app code**
  * The **application environment** (e.g., Node, Python)
  * A base **OS layer** (e.g., Linux)
* Containers do **not** store a copy of the image â€” they run on top of the image using:

  * A **thin, writable container layer**
  * Image layers remain **read-only and shared** between containers.

You may inspect an image to:

* Check **how it was built**
* See **exposed ports, env vars, entrypoint**
* Understand **layer structure** for performance or debugging
* Inspect external images you pulled (e.g., from Docker Hub)


# **Copying Files Into & From a Container**

### **Core Command: `docker cp`**

`docker cp` lets you **copy files or folders between your local machine and a running container**.

---

## **1. Copy FROM local → TO container**

**Syntax:**

```
docker cp <local-path> <container-name>:<container-path>
```

**Example:**
Copy a local folder `Dummy/` into a running container inside a new folder `/Test`:

```
docker cp Dummy/. mycontainer:/Test
```

* `Dummy/.` → copy all contents inside `Dummy`
* Destination folder is created automatically if it doesn't exist.

---

## **2. Copy FROM container → TO local**

**Syntax:**

```
docker cp <container-name>:<container-path> <local-path>
```

**Example:**
Copy `/Test` folder from the container back to your local `Dummy` folder:

```
docker cp mycontainer:/Test Dummy
```

Copy only a specific file:

```
docker cp mycontainer:/Test/test.txt Dummy
```

---

## **3. When This Is Useful**

### **Good uses**

* Extract log files generated inside the container
  Example:

  ```
  docker cp mycontainer:/var/log/app.log .
  ```
* Copy out any generated data you need to inspect.
* Copy in configuration files (e.g., web server config).

### **Not recommended**

Copying updated source code *into* a running container
→ Error-prone, inconsistent, can’t replace files currently in use (e.g., `server.js`).

→ Proper solution: use image rebuilds or bind mounts (explained later in the course).

---

## **Key Takeaway**

`docker cp` is a convenient tool for occasional file transfers in and out of running containers, especially logs and generated data, but **not** for ongoing development or code updates.

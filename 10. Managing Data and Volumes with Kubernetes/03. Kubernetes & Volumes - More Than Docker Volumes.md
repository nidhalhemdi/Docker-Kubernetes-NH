**State = important data created or used by your application that must not be lost.**

There are two major categories:

### **1. User-Generated Data (Must Never Be Lost)**

Examples:

* User accounts
* Uploaded files
* Saved stories (like the `/story` example)

Usually stored in:

* Databases
* Files on disk

### **2. Intermediate / Temporary Data**

Examples:

* Cached results
* Temp files
* Data in memory
* Temporary DB tables

This type may not need long-term persistence, but is still important while the app runs.

### **With Kubernetes**

You **do NOT** start containers yourself.

Kubernetes:

* Schedules Pods
* Starts containers inside those Pods
* Decides where Pods run (which Node)
* Restarts or replaces Pods automatically

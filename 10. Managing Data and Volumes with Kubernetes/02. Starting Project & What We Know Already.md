# **Starting Project & What We Know Already**

## ‚úÖ **Overview**

This lecture introduces the **example Node.js project** used to explain volumes in Kubernetes. The app stores text in a file, so it helps demonstrate **data persistence**.

---

## üìå **What the Example App Does**

A simple **Node.js API** with:

### **Routes**

1. **GET /story**

   * Reads `story/text.txt`
   * Returns stored text

2. **POST /story**

   * Expects JSON: `{ "text": "something" }`
   * If text is not empty ‚Üí append to `story/text.txt`

### **Important**

* Data is written into:

  ```
  story/text.txt
  ```
* The **story folder** must persist ‚Üí this is where volumes come in.

---

## üê≥ **Docker Setup Provided**

The instructor provides:

* `Dockerfile` ‚Üí builds the Node app image
* `docker-compose.yml` ‚Üí runs the container + defines a **volume**

This lets the app persist stored story data **even if the container is recreated**.

---

## üîß **Useful Commands from the Lecture**

### **Start the app with Docker Compose**

```sh
docker compose up -d --build
```

* `-d` ‚Üí run in background
* `--build` ‚Üí rebuild image if needed

### **Test the API**

Use Postman or any API tool:

**GET request:**

```
GET http://localhost:3000/story
```

**POST request:**
Send JSON:

```json
{
  "text": "My text"
}
```

### **Stop & remove the container**

```sh
docker compose down
```

### **Remove all stopped containers (optional cleanup)**

```sh
docker container prune
```

(type `y` to confirm)

### **Re-run the app**

```sh
docker compose up -d
```

‚û°Ô∏è **The old story text is still there** because the volume preserved it.

## üß† **What This Lecture Wants You to Remember**

* You already know how **Docker volumes** preserve data across container restarts/removal.
* The Node app is intentionally simple to highlight this concept.
* The next step is learning **how this idea translates to Kubernetes**.
* Kubernetes pods are **ephemeral**, so we need:

  * **Volumes**
  * **Persistent Volumes (PV)**
  * **Persistent Volume Claims (PVC)**

This lecture sets up the base example that you will migrate to Kubernetes storage concepts.

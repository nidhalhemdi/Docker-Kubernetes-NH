### **2. Nodes**

A **node** is a physical or virtual machine in the cluster.

There are **two types**:

#### **a. Master Node (Control Plane)**

Responsible for:

* Managing the cluster
* Deciding where pods go
* Ensuring everything is healthy

Runs components such as:

* API Server
* Scheduler
* Controller Manager

#### **b. Worker Nodes**

These are the machines that **actually run your application**.
Each worker node hosts:

* Pods
* Containers inside those pods
* Resources needed by those containers

---

### **3. Pods**

A **pod** is the smallest deployable unit in Kubernetes.
It is basically a **wrapper** around one or more containers and their resources.

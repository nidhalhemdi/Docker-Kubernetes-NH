# **Module Introduction**

---

## ✅ **What This Module Is About**

This module moves from Kubernetes theory → **actual hands-on work**.
You will **set up Kubernetes locally**, create and manage **Kubernetes objects**, and deploy a **real basic app**.

---

## ✅ **Key Ideas**

### **Kubernetes Testing Environment**

A local setup where you can run a small Kubernetes cluster.
Examples:

* **Minikube**
* **Docker Desktop Kubernetes**
  This lets you test everything on your machine before deploying to the cloud.

### **Kubernetes Objects**

Everything you create/configure in Kubernetes is an *object*.
Examples you’ll learn in this module:

* **Deployment** → manages running app instances (Pods)
* **Pod** → smallest unit, runs containers
* **Service** → exposes Pods to the network
* **ConfigMap / Secret** → store configuration
* **Ingress** → routing from outside world
  Kubernetes uses YAML files to define these objects.

### **Deployment**

A Kubernetes object that ensures a certain number of Pods are running.
If a Pod crashes → the Deployment recreates it automatically.

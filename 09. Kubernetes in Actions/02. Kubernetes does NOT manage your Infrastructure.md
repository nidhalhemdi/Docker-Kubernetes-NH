# **Kubernetes does NOT manage your Infrastructure**

---

# **Core Idea**

**Kubernetes manages your *containers*, NOT your servers.**
You must create and manage the underlying machines (nodes) yourself â€” unless you use an external tool or a managed cloud service.

---

# **What Kubernetes *Does***

Kubernetes is responsible for **orchestrating containers**, specifically:

* Running containers inside **Pods**
* Monitoring containers
* Restarting failed Pods
* Scaling Pods up or down
* Scheduling Pods across worker nodes
* Ensuring apps remain accessible

ğŸ‘‰ Kubernetes **uses** the resources you provide (nodes), but **does not create or manage** those resources.

---

# âŒ **What Kubernetes Does *NOT* Do**

Kubernetes will **NOT**:

* Create servers, VMs, or cloud instances
* Install the Kubernetes control plane on those servers
* Install kubelets on worker nodes
* Configure networks, security groups, load balancers
* Update or secure operating systems
* Manage data centers or server hardware

ğŸ‘‰ These tasks remain **your responsibility** or the responsibility of a separate tool.

---

# **Your Job (If No Managed Service Is Used)**

You must:

1. **Create master + worker nodes**

   * Physical machines or cloud VMs (AWS EC2, etc.)
2. **Install Kubernetes components**

   * API server (master)
   * Scheduler, controller manager
   * kubelet (workers)
   * kube-proxy
3. **Set up networking + security**
4. **Maintain and update OS + cluster**

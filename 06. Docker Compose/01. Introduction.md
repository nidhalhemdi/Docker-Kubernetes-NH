### **1. Multi-Container Apps Are Hard to Manage Manually**

In the last module, we built a 3-container application:

* **MongoDB database**
* **Node.js/Express backend**
* **React frontend**

Each container required:

* Long `docker run` commands
* Environment variables
* Volumes
* Networks
* Port mappings
* Building images manually
* Stopping/removing containers manually

**Running everything manually takes too much time and is error-prone.**

---

### **2. Real Projects Often Need Even More Containers**

If 3 containers already require many commands,
imagine:

* Caching layers (Redis)
* Messaging systems (RabbitMQ)
* Auth services
* Workers
* Proxies

This becomes unmanageable without automation.

---

### **3. Docker Compose Solves This**

Docker Compose is a tool that:

* Lets you **define all containers + config in a single `docker-compose.yml` file`**
* Starts everything with **one command**
* Stops everything with **one command**
* Handles networks, volumes, dependencies automatically
* Makes local development **much faster and cleaner**

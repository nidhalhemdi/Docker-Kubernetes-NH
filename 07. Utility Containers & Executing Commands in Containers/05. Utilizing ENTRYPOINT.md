# ðŸ”¥ **Why This Is NOT the Same as `CMD`**

### CMD gets *overwritten*:

If your Dockerfile had

```dockerfile
CMD ["npm"]
```

and you run:

```
docker run myimage install
```

Docker will **replace CMD**, producing:

```
install
```

(not what you want)

### ENTRYPOINT gets *appended*:

With

```dockerfile
ENTRYPOINT ["npm"]
```

and you run:

```
docker run myimage install
```

Docker executes:

```
npm install
```

â€”exactly what you want.

# ðŸ“¦ **Why Bind Mount + ENTRYPOINT Is Useful**

Your workflow becomes **super easy**:

1. Bind mount your project into `/app`
2. Use `ENTRYPOINT` to always run `npm`
3. Only type the subcommand (`install`, `init`, etc.)

This removes the need to install Node/npm locally,
and keeps everything isolated in a container.

# ğŸ“Œ **Key Concepts (Easy Definitions)**

### **1. `docker ps`**

* Shows **running** containers.
* By default, *stopped containers are hidden*.

### **2. `docker ps -a`**

* Shows **all** containers: running + stopped.
* Useful to find containers you might want to restart.

### **3. `docker run`**

* Creates **a brand-new container** from an image.
* Starts it **attached** to your terminal (blocking mode).
* Use when:

  * You changed the code
  * You rebuilt the image
  * You want a fresh container instance

### **4. `docker start`**

* **Restarts an existing container** (does NOT create a new one).
* Runs in **detached mode** (does NOT block the terminal).
* No logs show unless you explicitly request them.

### **5. Why restart instead of run?**

* If your **image or code has not changed**, there's **no need** to create a new container.
* Save time. Reuse the previous container.

* Use **`docker run`** only when:

  * You built a **new image**
  * Code or dependencies have changed
* Use **`docker start`** when:

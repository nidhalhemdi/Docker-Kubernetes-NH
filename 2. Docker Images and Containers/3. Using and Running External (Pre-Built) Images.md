## ðŸ”¹ Running an External Image

To use a pre-built image, simply run:

```bash

```

Example:

```bash

```

What happens:

* Docker checks for the image **locally**
* If not found â†’ Docker **pulls** it from Docker Hub automatically
* Then Docker creates a **container** *based on that image*

## ðŸ”¹ Verifying Containers

See all containers (running or exited):

```bash
docker ps -a
```

Output shows:

* Container ID
* Image used
* Creation time
* Status (e.g., **Exited**)
* Auto-generated name

This confirms containers were created even if you saw nothing happen.

## ðŸ”¹ Running a Container with Interactive Access

To open an interactive session **inside** the container:

```bash

```

Meaning of flags:

* `-i` â†’ interactive (keeps STDIN open)
* `-t` â†’ pseudo-terminal
  Together: **interactive terminal access**

Inside the container, you now have the Node REPL:

```bash

2
```

## ðŸ”¹ Exiting the Container

Exit interactive mode:

* Press **Ctrl + C twice**
* Or type:

  ```bash
  .exit
  ```

Once you exit:

* The container stops
* It still shows up in `docker ps -a` as **Exited**

## ðŸ”¹ Multiple Containers From the Same Image

You can create and run multiple containers from the **same** image:

```bash
docker run -it node
docker run -it node
```

Each one:

* Is a separate instance
* Runs independently
* Shares the same underlying image

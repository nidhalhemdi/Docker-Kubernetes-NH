# **Naming & Tagging Containers & Images**

### **1. Naming Containers**

* By default, Docker assigns random container names (e.g., *elegant_butterfly*).
* You can **assign your own container name** when running it:

  ```
  docker run --name goalsapp ...
  ```
* Benefits:

  * Easier to remember & use.
  * Easier to stop/remove without listing all containers:

    ```
    docker stop goalsapp
    ```

---

### **2. Tagging Images**

* Images have:

  * **Repository (name)** → general group (e.g., `node`, `goals`)
  * **Tag** → specific version/configuration (e.g., `14`, `latest`, `v1`)
  * Combined as:

    ```
    name:tag
    ```

* Example:

  ```
  node:14
  node:12
  node:slim
  ```

  These are different versions/configurations of the *node* image.

---

### **3. Tagging Your Own Images**

* You add a name + tag when building an image using `-t`:

  ```
  docker build -t goals:latest .
  ```

* You choose any tag you want:

  * `latest`
  * `v1`
  * `production`
  * etc.

* Viewing images:

  ```
  docker images
  ```

  You’ll see:

  ```
  REPOSITORY   TAG
  goals        latest
  ```

---

### **4. Running Images by Name:Tag**

Instead of using the long image ID, you can run containers using the tag:

```
docker run goals:latest
```

---

### **5. Why Tags Matter**

* Let you create **multiple versions** of an image.
* Useful when:

  * Deploying different versions
  * Trying new configurations
  * Using specific image versions (e.g., Node 12 vs Node 14)

### **1. What is a Docker Image?**

* A **template / blueprint** for creating containers.
* Contains:

  * OS layer(s)
  * Application code
  * Dependencies
* You **donâ€™t run an image directly** â€” you run a container *from* an image.

### **2. What is a Docker Container?**

* A **running instance** of an image.
* Basically:
  **Image = recipe**
  **Container = the meal created from that recipe**
* Containers are isolated but lightweight.

### **3. Relationship Between Images and Containers**

* You **must have an image** to create a container.
* A single image can create many containers.
* Containers depend on images for:

  * How they run
  * What is installed
  * How software behaves

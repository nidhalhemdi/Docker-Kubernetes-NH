# **Pulling & Using Shared Images**

### **Core Idea**

Once an image is pushed to Docker Hub (or any registry), anyone can **pull** and **run** it. This lecture explains how pulling works and how Docker decides whether to download a new image or use a cached one.

---

## **Key Concepts & Commands**

### **1. Pulling an Image**

* To download an image from Docker Hub (public image):

  ```bash
  docker pull <username>/<image-name>
  ```

  Example:

  ```bash
  docker pull nidhalhaamdii/node-hello-world
  ```
* If no tag is specified, Docker defaults to `:latest`.

### **2. Running a Pulled Image**

* After pulling:

  ```bash
  docker run <username>/<image-name>
  ```
* If the image **is not present locally**, `docker run` will automatically **pull it first**, then run it.

---

## **3. Docker Pull Behavior**

### **Important rules:**

#### **Rule 1 — Docker Pull always fetches the latest version**

If you run:

```bash
docker pull nidhalhaamdii/node-hello-world
```

Docker will always download the **newest image** from the registry.

#### **Rule 2 — Docker Run will NOT check for updates**

If an image already exists locally:

```bash
docker run nidhalhaamdii/node-hello-world
```

Docker **won’t check** if a newer version exists on Docker Hub.

To update your local copy, you must manually run:

```bash
docker pull nidhalhaamdii/node-hello-world
```

---

## **4. Public Images**

* Anyone can pull public images without logging in.
* Upload (push) requires authentication, but pull does not.

---

## **5. Example Workflow**

```bash
docker image prune -a      # Remove all images
docker pull nidhalhaamdii/node-hello-world
docker run -p 8000:8000 nidhalhaamdii/node-hello-world
docker stop <container-name>
```
